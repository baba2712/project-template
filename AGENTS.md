# AI Agent Instructions

This repository is intended to be used as a reusable project template. These instructions apply to all AI coding agents working in this repository.

## Mission
- Keep the template simple, reusable, and well documented.
- Favor maintainability over novelty.
- Make the smallest change that fully solves the task.

## Working rules
- Read the repository context before making broad changes.
- Follow instructions in `.github/copilot-instructions.md` and `.github/instructions/` when relevant.
- Keep source code in `src/`, tests in `tests/`, and docs in `docs/`.
- Update README or docs when setup, structure, or workflows change.
- Do not introduce unnecessary dependencies or tooling.

## Code changes
- Prefer explicit, readable implementations.
- Preserve backward compatibility unless a change intentionally updates the template contract.
- If creating new files, place them in the most appropriate standard directory.

## Documentation changes
- Keep examples generic.
- Explain assumptions clearly.
- Use concise language and actionable guidance.

## Testing expectations
- Add or update tests when behavior changes.
- Prefer deterministic tests.
- Reuse existing patterns if present.

## Collaboration
- Summarize what changed and why.
- Call out follow-up work when appropriate.
- If information is missing, make the safest reasonable assumption and document it.
